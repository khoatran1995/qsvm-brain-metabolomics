## 🔬 Methodology

### 1. Data Preparation
- **Dataset:** Metabolomics data from mouse brain samples predicting Sex (Male vs Female).
- **Samples:** 100 total (80 Training, 20 Testing).
- **Features:** 5 principal components (reduced via PCA to fit within hardware and qubit constraints).

### 2. Quantum Configuration
- **Feature Map:** `PauliFeatureMap` with `['Y', 'Z']` gates, 1 repetition, and 5 qubits.
- **Algorithm:** `QSVC` (Quantum Support Vector Classification) from `qiskit_machine_learning`.
- **Fidelity:** `ComputeUncompute` with a custom sampler.

### 3. Simulation & Feature Map Exploration
Before moving to real hardware, an extensive simulation phase was conducted to find the optimal data and quantum configuration:
* **Feature Map Exploration:** We tested different feature mapping strategies (including `ZFeatureMap`, `ZZFeatureMap`, and `PauliFeatureMap`) to see which one best captured the non-linear relationships in the data.
* **Hyperparameter Tuning:** We experimented with different sample sizes and PCA feature dimensions to find the configuration that maximized the quantum performance.
* **The "Sweet Spot":** This exploration led us to the combination of **100 samples and 5 features**, which yielded a highly competitive 95% accuracy on the simulator, making it the perfect candidate for physical hardware validation.

### 4. Hardware Execution & Problem Solving
Running on real hardware presented several challenges that were successfully overcome:

#### A. The `sxdg` ISA Error (Instruction Set Architecture)
* **Problem:** The `ComputeUncompute` method inverts the feature map to calculate fidelity. This generated `sxdg` gates which the real device (`ibm_fez`) does not support natively, causing the job to crash.
* **Solution:** Developed a custom `HardwareSampler` class that intercepts the circuits generated by the library and forces them to transpile correctly for the specific backend's supported gate set right before submission.

#### B. The 10 Million Shot Limit
* **Problem:** Comparing 80 training samples requires calculating $80 \times 80 = 6,400$ kernel entries. With default high shot counts, the total requested shots exceeded IBM's limit of 10,000,000 shots per job.
* **Solution:** Injected `kwargs.setdefault('shots', 100)` into the custom sampler's `run` method to keep the total workload well within the system limits while maintaining sufficient precision.

## 💻 Code Highlight: The Hardware Fix

Here is the custom wrapper that made the real hardware execution possible with Qiskit 1.0+ constraints:

```python
from qiskit_ibm_runtime import SamplerV2
from qiskit import transpile

class HardwareSampler(SamplerV2):
    def __init__(self, backend):
        super().__init__(mode=backend)
        self.backend = backend
        
    def run(self, pubs, **kwargs):
        transpiled_pubs = []
        for pub in pubs:
            circuit, values = pub
            # Transpile the circuit to match the hardware ISA
            transpiled_circuit = transpile(circuit, backend=self.backend, optimization_level=3)
            transpiled_pubs.append((transpiled_circuit, values))
        
        # Set shots to 100 to avoid exceeding the system limit!
        kwargs.setdefault('shots', 100)
        return super().run(transpiled_pubs, **kwargs)
```

## 📈 Scale and Noise Test (150 Samples)
To test the scalability of the model and observe the effect of physical quantum noise, we scaled the dataset to **150 samples** (120 training, 30 testing) and compared the simulator against the real hardware:

| Environment | Classical SVM | Quantum SVM |
| :--- | :--- | :--- |
| **Simulator** | 96.67% | 96.67% |
| **Real Hardware** | 96.67% | **93.33%** |

### Observations:
1. **Simulation Success:** In the perfect, noiseless simulation, the Quantum SVM tied with the Classical SVM at **96.67%** accuracy, showing that the feature map scales well!
2. **Noise Impact:** Running on real `ibm_marrakesh` hardware (127-qubit) caused a small drop to **93.33%** accuracy — a direct, measurable observation of quantum decoherence and gate errors affecting classification boundaries at scale.
